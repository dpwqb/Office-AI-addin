Office AI addin

Office AI addin 是适配 Microsoft Office 的 AI 插件，支持 Excel、Word、PowerPoint 三大应用，以侧边栏集成和展示， 通过自然语言交互赋能文档工作流。插件会自动识别当前所在的 Office 应用， 加载对应的工具集——在 Excel 中处理数据与公式、在 Word 中撰写与排版、 在 PowerPoint 中生成与整理幻灯片。

产品概述

Office AI addin 是适配 Microsoft Office 官方的 AI 插件，覆盖 Excel / Word / PowerPoint，以侧边栏集成和展示， 通过自然语言交互赋能文档工作流。一套插件按宿主自适应，分别实现 表格数据处理、文档撰写排版、演示文稿生成等任务的自动化执行。

核心由 GPT 模型、Agent SDK/Agent loop、Skills 等模块驱动， 适配 Microsoft 365 的 Office 环境，其核心交互与能力如下：

  • 自然语言驱动： 在侧边栏输入指令（如 Excel 中“制作可视化图表”“修复 #REF! 错误”、 Word 中“在文末续写一段总结”、PowerPoint 中“新增一页幻灯片”）， LLM 直接解析并操作文档，生成可追溯的内容级引用。
  • 闭环执行流程： Agent 拆解任务 → 调用相应的 tools → Agent Loop 校验结果 → 在当前 Office 应用中直接修改并定位变更，全程不离开文档环境。
  • 宿主自适应： 插件根据当前所在的 Excel / Word / PowerPoint 自动切换工具集与提示词， 无需用户手动选择。

各宿主典型能力：

Excel： 图表生成、公式错误检测与修复、跨 Sheet 导航 / VLOOKUP、数据读写与透视。

Word： 大纲解析、撰写与续写、查找替换、套用标题样式、插入表格 / 图片、批注。

PowerPoint： 演示大纲解析、增删幻灯片、设置讲稿备注、插入文本框、跳转定位。

优势亮点

长上下文深度解析

依托 LLM 大上下文，支持十万级单元格与跨表 / 跨文件依赖解析， 提供单元格级引用溯源。

精准公式与排错

自然语言生成 / 修改公式，诊断 #REF!、 #VALUE!、 循环引用等并给修复方案， 更新假设时保留公式依赖，降低人工校验成本。

工具调用（Tool Use）

一次指令同时触发 「读取数据→分析结构→选择图表类型→配色→插入图表」 多步操作链，体现模型的多步规划与执行能力（Agentic Planning）。

语义级理解（Semantic Understanding）

不是简单的「数据→图」映射，而是理解 「厂商」是分类维度、 「ELO分数」是排序指标、 「投票数」是置信度指标， 从而自主决定图表类型和配色策略。

跨上下文结构理解（Cross-context Structure Parsing）

模型需要同时理解 3 个 Sheet 的表结构， 识别「产品编码」是跨表连接键， 判断哪些字段该用 VLOOKUP（一对一查询）、 哪些该用 SUMIF（一对多汇总）。 这要求模型具备数据建模级别的理解能力。

适用场景

企业财务（FP&A / 会计 / 审计）

  • 预算与预测： 多部门预算自动分摊，支持假设场景切换，生成动态预算表。
  • 财务审计： 追踪单元格引用，定位公式错误，生成审计线索，确保数据可追溯。
  • 数据治理： 批量清洗 ERP 导出数据，统一格式，自动识别异常值并标注。

零售与电商（运营 / 供应链）

  • 销售复盘： 整合多平台订单数据，生成区域 / 渠道 / 产品维度的可视化报表。
  • 库存优化： 分析历史消耗数据，计算安全库存阈值，生成补货公式与预警。
  • 促销效果模拟： 模拟价格 / 折扣调整对利润影响，输出敏感性分析与瀑布图。

咨询与专业服务

  • 客户数据中台： 合并多源 Excel 数据，按客户 ID 对齐，生成行业对标看板。
  • 项目交付提效： 批量处理客户报表，自动生成 PPT 可用的图表与结论，缩短交付周期。
  • 模型复用： 将方法论固化为 Excel 模板，用自然语言一键生成定制化分析报告。

制造业与供应链

  • 生产排程优化： 基于订单 / 产能数据，生成最优排程表，支持多场景模拟。
  • 成本核算： 自动分摊直接 / 间接成本，生成成本分析表，定位降本空间。
  • 供应链风控： 分析供应商数据，识别交付风险，生成风险评级与备选方案。

文档撰写与编辑（Word）

  • 报告与公文： 按大纲撰写 / 续写正文，统一标题层级与样式，一键插入数据表格。
  • 校对润色： 全文查找替换、修正措辞语法，关键处插入批注供复核。
  • 结构提炼： 解析文档大纲，生成摘要与结论，定位跳转到指定段落。

演示文稿制作（PowerPoint）

  • 快速成稿： 按主题新增幻灯片、插入文本框与要点，整理演示结构。
  • 讲稿备注： 为每页自动生成讲稿备注，辅助演讲准备。
  • 大纲梳理： 读取演示大纲、精炼要点文字，跳转定位到指定幻灯片。

配置步骤

先选择目标应用

每个 Office 应用需要使用各自的清单文件（manifest）。请根据要安装到的应用下载对应文件， 再按下方步骤旁加载。三套清单指向同一插件，安装方式完全一致，只是文件不同：

↓ Excel：manifest.excel.prod.xml ↓ Word：manifest.word.prod.xml ↓ PowerPoint：manifest.ppt.prod.xml

下文步骤以 Excel + manifest.excel.prod.xml 为例； 安装到 Word 或 PowerPoint 时，将文件名替换为 manifest.word.prod.xml 或 manifest.ppt.prod.xml， 并把对应的应用、Wef 目录与信任中心操作改为目标应用即可。

第一步：确认已安装目标 Office 应用

在开始安装插件前，请确保：

  • 电脑已安装 目标 Office 应用（Excel / Word / PowerPoint，Microsoft 365 或 Office 2016 及以上版本）
  • 该应用可以正常打开
  • 当前账户具有本机文件读写权限

如果尚未安装，请先安装 Microsoft Office 后再继续以下步骤。

第二步：下载清单文件

下载与目标应用对应的清单文件（见本节顶部下载区）： Excel 用 manifest.excel.prod.xml、 Word 用 manifest.word.prod.xml、 PowerPoint 用 manifest.ppt.prod.xml。

下载完成后请确认：

  • 文件名称与目标应用一致（如 manifest.excel.prod.xml）
  • 文件后缀为 .xml
第三步：将文件放入 Wef 文件夹并开启共享

1. 打开 Wef 文件夹

按下： Win + R

输入（Windows 上 Excel / Word / PowerPoint 共用同一 Wef 目录）： %LOCALAPPDATA%\Microsoft\Office\16.0\Wef

2. 放入清单文件

将下载好的清单文件（如 manifest.excel.prod.xml） 复制到： Wef 文件夹根目录

注意事项
  • 不要放入子文件夹
  • 文件后缀必须为 .xml
  • 不要修改文件名

3. 配置共享

  1. 右键点击 Wef 文件夹
  2. 选择 属性
  3. 切换到 共享 选项卡
  4. 点击：共享(S)... → 添加
  5. 将权限级别改为：读取/写入
  6. 点击：共享

共享成功后，会显示类似如下的网络路径： \\你的电脑名\…\Wef

请记录该路径，后续需要使用。

第四步：在目标应用中信任共享路径

完成共享后，需要在目标 Office 应用（Excel / Word / PowerPoint）中 将该网络路径添加为“受信任的加载项目录”。

1. 打开信任中心

在目标应用中依次点击： 文件 → 选项 → 信任中心 → 信任中心设置(T)…

2. 添加受信任的加载项目录

  1. 在左侧选择：受信任的加载项目录
  2. 在 “目录 URL(U)” 输入框中填写共享路径
  3. 点击：添加目录(D)
  4. 勾选：在菜单中显示
  5. 点击确定保存设置

示例路径： \\DESKTOP-XXXX\…\Wef

提示
信任中心的设置按应用独立。若要在多个 Office 应用中使用插件， 需在每个应用里分别添加同一条受信任目录。
第五步：在目标应用中加载插件
  1. 完全退出目标应用
  2. 重新打开后：进入 文件 → 选项 → 自定义功能区
  3. 找到 我的加载项 将其添加到选项卡
  4. 打开选项卡，点击：我的加载项 的下拉项
  5. 找到插件
  6. 点击：添加

插件加载完成后，即可在目标应用中使用。插件会自动识别当前应用并加载对应工具集。

在开始安装插件前，请确保：

  • 已安装 目标 Office for Mac 应用（Excel / Word / PowerPoint，Microsoft 365 或 Office 2019 及以上版本）
  • 该应用可以正常启动
  • 当前用户账户具有文件读写权限

  • 文件名称与目标应用一致（如 manifest.excel.prod.xml）
  • 文件后缀为 .xml
第三步：打开 Mac 上的 Wef 文件夹

在 macOS 上，每个 Office 应用有各自独立的 Wef 目录。请根据目标应用选择对应路径：

  • Excel：~/Library/Containers/com.microsoft.Excel/Data/Documents/wef/
  • Word：~/Library/Containers/com.microsoft.Word/Data/Documents/wef/
  • PowerPoint：~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/

如未找到 wef 文件夹，请按以下步骤打开目录并创建：

  1. 打开 Finder
  2. 点击顶部菜单 前往 → 前往文件夹…
  3. 输入对应应用的容器路径（去掉末尾 wef/），例如： ~/Library/Containers/com.microsoft.Excel/Data/Documents/
  4. 若没有 wef 文件夹，请新建一个名为 wef（全小写）的文件夹
第四步：复制 manifest 文件

将下载好的清单文件（如 manifest.excel.prod.xml）

复制到： 对应应用的 wef 文件夹根目录

注意事项
  • 放入与目标应用匹配的容器 wef 目录（Excel / Word / PowerPoint 各自独立）
  • 不要放入子文件夹
  • 不要修改文件名
  • 文件后缀必须为 .xml

  2. 重新打开后：进入 文件 → 选项 → 自定义功能区

  4. 打开选项卡，点击：我的加载项 的下拉项
